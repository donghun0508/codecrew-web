# 모각코 서비스 PRD (Product Requirements Document)

## 1. 개요

### 1.1 제품명
**모각코 (CodeCrew)** - 모여서 각자 코딩하는 온라인 스터디 플랫폼

### 1.2 제품 비전
개발자들이 온라인에서 함께 모여 각자의 프로젝트를 진행하며, 동기부여와 생산성을 높일 수 있는 협업 공간을 제공합니다.

### 1.3 목표
- 혼자서는 집중하기 어려운 개발자들에게 함께하는 환경 제공
- 실시간 소통을 통한 즉각적인 피드백 및 협업 촉진
- 학습 및 개발 활동의 지속성과 책임감 향상
- 개발자 커뮤니티 형성 및 네트워킹 기회 제공

### 1.4 타겟 사용자
- 혼자 공부/개발하기 어려운 주니어 개발자
- 사이드 프로젝트를 진행하는 개발자
- 온라인 스터디 그룹을 운영하거나 참여하는 개발자
- 재택근무 중 동료와의 연결감이 필요한 개발자

## 2. 핵심 기능 요구사항

### 2.1 세션 관리

#### 2.1.1 세션 생성
- 공개/비공개 세션 생성 가능
- 세션 정보 입력
  - 세션명
  - 세션 설명
  - 예상 시간
  - 최대 참여 인원 (2~10명)
  - 세션 태그 (언어, 주제, 난이도 등)
- 반복 세션 설정 (일회성/매일/매주)

#### 2.1.2 세션 참여
- 실시간 진행 중인 세션 목록 조회
- 필터링 및 검색
  - 태그별 필터
  - 참여 가능 인원
  - 진행 시간대
- 세션 참여 신청 및 즉시 입장
- 초대 링크를 통한 참여

#### 2.1.3 세션 진행
- 실시간 참여자 목록 표시
- 호스트 권한 관리
  - 참여자 관리 (강퇴, 음소거 등)
  - 세션 종료 권한
- 타이머 기능
  - 포모도로 타이머 (25분 작업 + 5분 휴식)
  - 커스텀 타이머 설정
  - 공동 타이머 공유

### 2.2 실시간 커뮤니케이션

#### 2.2.1 화상/음성 통화
- WebRTC 기반 실시간 연결
- 음성 전용 모드 지원
- 개별 음소거 기능
- 화면 공유 기능
- 음성/영상 품질 조절

#### 2.2.2 채팅
- 텍스트 채팅
- 코드 스니펫 공유 (syntax highlighting)
- 링크 공유 및 미리보기
- 이모지 반응
- 채팅 내역 저장

### 2.3 개인 활동 관리

#### 2.3.1 목표 설정
- 세션별 목표 설정
- 체크리스트 작성
- 진행 상황 업데이트
- 목표 달성률 추적

#### 2.3.2 활동 기록
- 참여 세션 히스토리
- 총 학습/개발 시간 통계
- 일일/주간/월간 리포트
- 연속 참여 일수 (Streak)

#### 2.3.3 업적 시스템
- 참여 시간 기반 뱃지
- 연속 참여 뱃지
- 목표 달성 뱃지
- 커뮤니티 기여 뱃지

### 2.4 사용자 프로필

#### 2.4.1 프로필 정보
- 닉네임 및 프로필 이미지
- 한줄 소개
- 주요 기술 스택
- GitHub/LinkedIn 연동
- 활동 통계 표시

#### 2.4.2 설정
- 알림 설정
- 화상/음성 설정
- 테마 설정 (라이트/다크 모드)
- 개인정보 보호 설정

### 2.5 커뮤니티

#### 2.5.1 피드
- 완료된 세션 공유
- 학습 내용 공유
- 좋아요 및 댓글
- 팔로우 기능

#### 2.5.2 그룹
- 스터디 그룹 생성 및 관리
- 그룹 전용 세션
- 그룹 게시판
- 멤버 관리

## 3. 비기능적 요구사항

### 3.1 성능
- 페이지 로딩 시간: 3초 이내
- WebRTC 연결 지연: 500ms 이내
- 동시 접속자: 최소 1,000명 지원
- 세션당 최대 참여자: 10명

### 3.2 보안
- HTTPS 통신
- JWT 기반 인증
- OAuth 2.0 소셜 로그인
- 개인정보 암호화 저장
- CORS 정책 적용

### 3.3 호환성
- 브라우저: Chrome, Firefox, Safari, Edge (최신 2개 버전)
- 반응형 디자인 (Desktop, Tablet, Mobile)
- WebRTC 지원 브라우저

### 3.4 확장성
- 마이크로서비스 아키텍처 고려
- 수평적 확장 가능한 구조
- CDN을 통한 정적 자원 배포

### 3.5 접근성
- WCAG 2.1 AA 레벨 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환

## 4. 기술 스택 (제안)

### 4.1 프론트엔드
- Framework: React 18+ / Next.js
- State Management: Redux Toolkit / Zustand
- Styling: Tailwind CSS / Styled Components
- WebRTC: Simple-peer / PeerJS
- Real-time: Socket.io-client

### 4.2 백엔드
- Runtime: Node.js
- Framework: Express / NestJS
- Database: PostgreSQL (메인), Redis (캐시/세션)
- Real-time: Socket.io
- Authentication: Passport.js / JWT

### 4.3 인프라
- Cloud: AWS / GCP
- Container: Docker
- CI/CD: GitHub Actions
- Monitoring: Sentry, CloudWatch

## 5. 릴리스 계획

### 5.1 MVP (Phase 1) - 2개월
- 사용자 인증 (소셜 로그인)
- 세션 생성 및 참여
- 기본 화상/음성 통화
- 텍스트 채팅
- 간단한 타이머 기능

### 5.2 Phase 2 - 3개월
- 목표 설정 및 관리
- 활동 기록 및 통계
- 프로필 페이지
- 화면 공유 기능
- 알림 시스템

### 5.3 Phase 3 - 4개월
- 업적 시스템
- 커뮤니티 피드
- 그룹 기능
- 고급 필터링 및 검색
- 모바일 최적화

## 6. 성공 지표 (KPI)

### 6.1 사용자 지표
- DAU (Daily Active Users)
- MAU (Monthly Active Users)
- 사용자 재방문율
- 평균 세션 참여 시간

### 6.2 참여 지표
- 일일 생성 세션 수
- 세션당 평균 참여자 수
- 평균 세션 지속 시간
- 목표 달성률

### 6.3 성장 지표
- 신규 가입자 수
- 바이럴 계수 (K-factor)
- 추천을 통한 가입률
- 사용자 리텐션 (1주/1개월/3개월)

### 6.4 품질 지표
- 버그 발생률
- 평균 응답 시간
- WebRTC 연결 성공률
- 사용자 만족도 (NPS)

## 7. 리스크 및 대응 방안

### 7.1 기술적 리스크
**리스크**: WebRTC 연결 안정성 문제
- **대응**: TURN 서버 구축, 자동 재연결 로직, 음성 전용 폴백

**리스크**: 대규모 동시 접속 처리
- **대응**: 로드 밸런싱, 캐싱 전략, DB 최적화

### 7.2 비즈니스 리스크
**리스크**: 사용자 확보 어려움
- **대응**: 초기 커뮤니티 구축, 인플루언서 마케팅, 오픈 베타 운영

**리스크**: 경쟁 서비스와의 차별화
- **대응**: 개발자 특화 기능 강화, UX 최적화, 커뮤니티 중심 운영

### 7.3 운영 리스크
**리스크**: 부적절한 콘텐츠 및 행동
- **대응**: 신고 시스템, 자동 모니터링, 커뮤니티 가이드라인
